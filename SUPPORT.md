# Support

## Where to ask for help

- Bug reports: open a GitHub issue with reproduction steps and environment
  details.
- Feature requests: open a GitHub issue and explain the user problem first,
  not just the proposed implementation.
- Security issues: do not open a public issue; follow `SECURITY.md`.

## Good issue reports include

- PhotonTrust version or commit SHA.
- Python version and platform.
- Exact command, config, graph, or API request used.
- Expected behavior vs actual behavior.
- Relevant logs, stack traces, or output artifacts.

## Self-serve resources

- `README.md` for installation and quick starts.
- `docs/README.md` for documentation indexes.
- `configs/README.md` for runnable example configs.
- `scripts/README.md` for validation, release, and demo scripts.
- `examples/README.md` for Python and notebook examples.
