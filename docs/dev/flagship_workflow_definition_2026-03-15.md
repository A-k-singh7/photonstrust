# Flagship Workflow Definition (2026-03-15)

## Goal

Define the one workflow PhotonTrust should demonstrate repeatedly to customers,
investors, and partners because it best explains the product's value.

## Recommended Flagship Workflow

**Compare a baseline and candidate system, decide which one is acceptable,
capture approval, and export verifiable evidence.**

## Why This Should Be The Flagship

1. it is understandable outside the research community,
2. it naturally incorporates trust, evidence, and decision-making,
3. it showcases more differentiation than raw simulation alone,
4. it connects technical outputs to organizational action.

## User Story

A technical lead or reviewer wants to know whether a candidate photonic or
quantum system configuration is better than the current baseline, whether it is
safe to approve, and how to export that decision as a reviewable evidence pack.

## Workflow Steps

### Step 1: start from a seeded workspace

- open a project-backed workspace
- choose guided or power mode
- load a realistic scenario

### Step 2: execute or select runs

- run the baseline and candidate, or load them from the run registry
- show confidence, assumptions, and provenance context

### Step 3: compare the decision delta

- frame baseline vs candidate clearly
- show changed fields and semantic impact
- summarize blockers, cautions, and recommendation signal

### Step 4: certify the decision

- inspect readiness state
- capture approval and note
- review evidence and manifest data

### Step 5: export and verify

- open/export the packet
- publish the evidence bundle
- verify the published result

## Product Value Demonstrated

This single workflow demonstrates that PhotonTrust can:

1. simulate,
2. compare,
3. justify,
4. approve,
5. export,
6. verify.

That is the strongest product story currently available in the repo.

## Variants

### PIC variant

- build layout artifacts
- generate GDS
- run KLayout extraction
- then move into compare/certify evidence framing

### Orbit/satellite variant

- run orbit-aware scenario
- compare operational envelopes
- move into evidence-backed decision mode

## Required Product Quality Bar

The flagship workflow must be:

1. understandable in under five minutes,
2. visually polished,
3. deterministic enough for repeatable demos,
4. backed by real outputs and not just mocked summaries,
5. exportable as screenshots, packets, and evidence links.

## What To Polish Further

1. final copy and screenshots,
2. stronger onboarding around baseline/candidate framing,
3. clearer recommendation language,
4. cleaner customer-facing screenshots and video capture.
