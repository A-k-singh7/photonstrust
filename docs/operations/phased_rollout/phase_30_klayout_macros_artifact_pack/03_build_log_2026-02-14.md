# Phase 30 - Build Log - KLayout Macro Templates + Run Artifact Pack Contract

Date: 2026-02-14

## Summary
Implemented a trusted KLayout macro template (batch runnable) plus a deterministic "KLayout run artifact pack" wrapper + schema contract. The result is a provenance-grade evidence bundle for GDS-derived extraction and DRC-lite checks, while keeping KLayout optional.

## Code/Artifact Changes

### 1) KLayout macro template (repo-owned, batch mode)
- Added:
  - `tools/klayout/macros/pt_pic_extract_and_drc_lite.py`
    - Extracts:
      - ports from TEXT labels on label layer using `{label_prefix}:{node}:{port}`
      - routes from PATH shapes on waveguide layer (centerline points + width)
    - Produces:
      - `ports_extracted.json`
      - `routes_extracted.json`
      - `drc_lite.json` (DRC-lite status + issue list)
      - `macro_provenance.json`

### 2) Artifact pack wrapper (Python)
- Added:
  - `photonstrust/layout/pic/klayout_artifact_pack.py`
    - Runs the macro via the existing KLayout runner seam (`run_klayout_macro`)
    - Captures:
      - stdout/stderr logs
      - command line
      - SHA256 hashes of inputs and outputs
    - Writes:
      - `klayout_run_artifact_pack.json` (schema-bound contract)
    - Policy:
      - if KLayout missing and `allow_missing_tool=True`, emits a schema-valid pack with `status="skipped"` and a clear error message

### 3) External tool discovery hardening
- Updated:
  - `photonstrust/layout/pic/klayout_runner.py`
    - Adds support for `PHOTONTRUST_KLAYOUT_EXE`/`KLAYOUT_EXE` override
    - Adds best-effort Windows install-path discovery

### 4) Schemas
- Added:
  - `schemas/photonstrust.pic_klayout_run_artifact_pack.v0.schema.json`
  - `schemas/photonstrust.pic_ports.v0.schema.json`
  - `schemas/photonstrust.pic_routes.v0.schema.json`
  - `schemas/photonstrust.pic_klayout_drc_lite.v0.schema.json`

### 5) Tests
- Added:
  - `tests/test_klayout_artifact_pack_schema.py`
    - Always validates the artifact pack schema and ensures the pack manifest is written.
    - Optional integration path runs only if both:
      - KLayout executable is discoverable, and
      - `gdstk` is installed to generate a GDS fixture.
