## 1) Outcome

Implemented the Phase 13 web editor MVP (drag-drop graph authoring) and wired it
to the local FastAPI backend for:
- graph compilation (`/v0/graph/compile`)
- QKD preview execution (`/v0/qkd/run`)
- PIC netlist simulation (`/v0/pic/simulate`)

The web UI is intentionally **graph-first**: the graph JSON is the source of
truth and is what is compiled/executed by the Python engine.

## 3) Key Product / Trust Decisions

- The UI does **not** implement physics logic. It authors graph JSON and shows
  compiled/run artifacts returned by the backend (assumptions included).
- Security posture for MVP: the API server rejects `pic.touchstone_2port`
  requests (Touchstone file access is CLI-only by default).
- PIC multi-input circuits require explicit `circuit.inputs` to avoid ambiguous
  auto-detection. The PIC MZI template sets a default `inputs` list.
