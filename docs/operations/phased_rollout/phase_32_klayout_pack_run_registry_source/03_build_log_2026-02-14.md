# Phase 32 - Build Log - KLayout Pack: Run Registry Source Selection

Date: 2026-02-14

## Build Summary
Extended the KLayout pack endpoint + web UI so the KLayout artifact pack can be generated from any selected run in the Run Registry (Runs mode), by selecting a `.gds` artifact path from that run's manifest.

## Changes Shipped
- Backend
  - Updated `photonstrust/api/server.py` `POST /v0/pic/layout/klayout/run` to accept `source_run_id` (preferred) in addition to legacy `layout_run_id`.
  - Added safe default `.gds` selection logic when `gds_artifact_path` is omitted:
    - use `artifacts.layout_gds` if present
    - else if exactly one top-level `.gds` artifact exists, use it
    - else best-effort fallback to `layout.gds` if present in the run dir
    - else 400 with a clear error requesting `gds_artifact_path`
  - Wrote manifest inputs: `source_run_id`, `source_gds_artifact_path`, and `layout_run_id` (nullable for compatibility).
  - Improved Windows KLayout discovery in `photonstrust/layout/pic/klayout_runner.py` by including `%APPDATA%` in common install location checks (Roaming installs).

- Frontend
  - Updated `web/src/photontrust/api.js` `apiRunPicKlayoutPack` to support `sourceRunId` (maps to `source_run_id`) while preserving `layoutRunId` support.
  - Added Runs mode UI (Manifest tab) in `web/src/App.jsx`:
    - detects `.gds` artifacts from `selectedRunManifest.artifacts`
    - provides dropdown selection of `gds_artifact_path`
    - runs KLayout pack against the selected run + selected `.gds`
    - displays served artifact links and JSON result

- Tests
  - Updated `tests/api/test_api_server_optional.py` with a new hermetic API test covering `source_run_id` + explicit `gds_artifact_path`.
  - Forced KLayout seam to behave as "missing tool" in API tests (monkeypatch `find_klayout_exe`) so tests do not invoke external KLayout even if installed.
