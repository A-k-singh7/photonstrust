# Phase 34 - Invdesign Workflow Chaining - Build Log

Date: 2026-02-14

## Summary
Implemented a first-class chained PIC workflow run that orchestrates:
- invdesign -> layout build -> LVS-lite -> optional KLayout artifact pack -> SPICE export

The workflow produces a top-level run manifest with a deterministic `workflow_report.json` that references child run IDs.

## Changes (By File)

### Backend
- `photonstrust/photonstrust/api/server.py`
  - Added `POST /v0/pic/workflow/invdesign_chain`.
  - Executes the chain by calling existing endpoint functions (internal calls) so child runs are created with the same artifact contracts as standalone operations.
  - KLayout step is best-effort (optional seam): skipped when no `.gds` exists; errors do not fail the overall workflow.
  - Workflow run type: `pic_workflow_invdesign_chain`.
  - Workflow artifacts:
    - `workflow_report.json` (referencing child run IDs + served artifact relpaths).

### Web
- `photonstrust/web/src/photontrust/api.js`
  - Added `apiRunPicInvdesignWorkflowChain()` client wrapper for `/v0/pic/workflow/invdesign_chain`.
- `photonstrust/web/src/App.jsx`
  - Added `workflowResult` state.
  - Added "Run Full Workflow" button next to the existing inverse-design action.
  - Added a "Workflow Chain Result" section under the InvDesign tab with links to the workflow report and child run manifests.

### Tests
- `photonstrust/tests/test_api_server_optional.py`
  - Added `test_api_pic_invdesign_workflow_chain_writes_outputs`.
  - Hermetic posture: monkeypatches `photonstrust.layout.pic.klayout_runner.find_klayout_exe` to ensure KLayout is not invoked during tests.

## Notes / Decisions
- Workflow status is derived from the LVS-lite pass boolean (`pass` vs `fail`).
- The workflow run references child run IDs instead of copying all artifacts into a single directory, preserving existing artifact contracts and keeping the run registry consistent.
