# Phase 27 - Build Log (2026-02-14)

## Summary
Implemented a deterministic, sidecar-first PIC layout build seam plus LVS-lite mismatch reporting.

Key idea: make `ports.json` + `routes.json` the stable contract that always exists (CI-friendly), and treat GDS emission and KLayout runs as optional backends.

## Code Changes

### New modules
- `photonstrust/layout/pic/spec.py`
  - layout settings normalization
  - node `ui.position` parsing with deterministic unit scaling
- `photonstrust/layout/pic/build_layout.py`
  - deterministic placement (UI position or grid fallback)
  - deterministic Manhattan routing for optical edges
  - emits artifacts: `ports.json`, `routes.json`, `layout_provenance.json`
  - optional GDS emission via `gdstk` (when installed)
- `photonstrust/layout/pic/extract_connectivity.py`
  - extracts observed connectivity by snapping route endpoints to port markers
- `photonstrust/verification/lvs_lite.py`
  - expected vs observed connectivity mismatch summary report
- `photonstrust/layout/pic/klayout_runner.py`
  - optional external tool seam to run KLayout batch macros/scripts

### New schemas
- `schemas/photonstrust.pic_layout_build.v0.schema.json`
- `schemas/photonstrust.pic_lvs_lite.v0.schema.json`

### New tests
- `tests/test_pic_layout_build_and_lvs_lite.py`
  - schema validation for layout build and LVS-lite reports
  - pass case (expected connectivity matches extracted)
  - fail case (dangling route endpoint -> missing edge)
- `tests/test_klayout_runner_optional.py`
  - if KLayout is not on PATH, runner raises a clear error

## Notes / Limitations (Intentional in v0.1)
- Routing is a deterministic Manhattan L-shape per edge. This is not congestion-aware routing.
- Connectivity extraction is from sidecars (routes + ports), not from full polygonal extraction.
- `layout.gds` is emitted only when `gdstk` is installed (`photonstrust[layout]`).
- KLayout is integrated as a runner seam only. Foundry rule decks and full LVS extraction remain future phases.

## Why This Helps the Platform
- Makes layout integration deterministic and reviewable (text artifacts).
- Enables LVS-style mismatch summaries without requiring foundry tools in CI.
- Keeps open-core clean while still providing a direct path to KLayout/PDK workflows.
