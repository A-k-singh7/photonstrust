# Build Log

- Work item ID: PT-PHASE-16
- Title: OrbitVerify web runner v0.1 (config-first pass envelopes)
- Date: 2026-02-13

## Summary

Implemented a minimal OrbitVerify web-runner that:
- edits an Orbit pass envelope config in the web UI,
- executes the simulation via the local FastAPI server, and
- returns the same trust artifacts as CLI workflows (results JSON + HTML report paths, plus provenance).

## Code Changes

### Backend (FastAPI)
- `photonstrust/photonstrust/api/server.py`
  - Added `POST /v0/orbit/pass/run`.
  - Accepts `{ "config": <dict>, "output_root": <string optional> }`.
  - Compatibility: allows posting the config dict directly as the request body.
  - Executes `photonstrust.orbit.pass_envelope.run_orbit_pass_from_config(config, run_dir)`.
  - Returns:
    - `run_id`
    - `config_hash`
    - `output_dir`, `results_path`, `report_html_path`
    - `results` (parsed JSON from `results_path`)
    - API-level provenance (`photonstrust_version`, `python`, `platform`)

### Web UI
- `photonstrust/web/src/photontrust/api.js`
  - Added `apiRunOrbitPass(baseUrl, config, { outputRoot })`.
- `photonstrust/web/src/App.jsx`
  - Added a topbar `Mode` selector:
    - `Graph Editor` (existing)
    - `Orbit Pass` (new)
  - Orbit Pass mode:
    - center panel shows a JSON editor seeded with a pass-envelope template
    - topbar `Run` triggers `POST /v0/orbit/pass/run`
    - right panel shows:
      - returned artifact paths
      - full JSON response (including parsed results)

### Tests
- `photonstrust/tests/test_api_server_optional.py`
  - Added an orbit pass config fixture and a smoke test for `POST /v0/orbit/pass/run`:
    - uses `tmp_path` as `output_root`
    - asserts `results_path` and `report_html_path` exist
    - asserts parsed `results` has expected keys (`pass_id`, `band`)

## Security Posture Notes (v0.1)
- The Orbit pass endpoint only accepts a config dict; it does not implement file serving.
- Paths are returned for local inspection; hosting/auth/storage is out of scope for v0.1.
