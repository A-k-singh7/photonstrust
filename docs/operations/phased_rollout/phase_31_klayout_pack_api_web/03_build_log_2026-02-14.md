# Phase 31 - Build Log - KLayout Artifact Pack API + Web Integration

Date: 2026-02-14

## Summary
Integrated the Phase 30 KLayout artifact pack into the managed workflow surface:
- added an API endpoint to run the trusted KLayout macro template against a prior layout run's GDS, and
- added a web UI tab to trigger the run and review served artifacts (pack JSON, logs, extracted outputs).

## Code Changes

### Backend (FastAPI)
- Updated: `photonstrust/api/server.py`
  - Added import: `build_klayout_run_artifact_pack`
  - Added endpoint: `POST /v0/pic/layout/klayout/run`
    - Input: `layout_run_id`, optional `settings`, optional `project_id`
    - Resolves GDS safely from the referenced layout run directory
    - Creates a new run with:
      - `klayout_run_artifact_pack.json`
      - `klayout_stdout.txt`, `klayout_stderr.txt`
      - macro outputs when present (`drc_lite.json`, `ports_extracted.json`, etc.)
    - Writes `run_manifest.json` with `run_type=pic_klayout_artifact_pack`

### Frontend (Web UI)
- Updated: `web/src/photontrust/api.js`
  - Added: `apiRunPicKlayoutPack(...)`
- Updated: `web/src/App.jsx`
  - Added PIC-mode tab: `KLayout`
  - Added state for KLayout pack settings/result
  - Added served artifact links for pack/logs/outputs

### Tests
- Updated: `tests/api/test_api_server_optional.py`
  - Added a hermetic test for `/v0/pic/layout/klayout/run` using a synthetic prior layout run containing a dummy `layout.gds`.
