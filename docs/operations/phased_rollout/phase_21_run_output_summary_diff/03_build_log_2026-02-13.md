## Changes Implemented

### Backend (API)
- Added stable, domain-aware `outputs_summary` blocks to run manifests:
  - QKD runs (`POST /v0/qkd/run`) now write:
    - `outputs_summary.qkd.cards[]` with `scenario_id`, `band`, `key_rate_bps`, `qber`, `safe_use`
  - Orbit pass runs (`POST /v0/orbit/pass/run`) now write:
    - `outputs_summary.orbit_pass.cases[]` with per-case integrated performance summaries
- Extended run diff endpoint to support output comparisons:
  - `POST /v0/runs/diff` now supports `scope`:
    - `input` (default)
    - `outputs_summary`
    - `all` (full manifest diff; bounded)

## Notes
- `outputs_summary` is intentionally small and stable. The goal is a review-friendly
  diff target that avoids diffing large result payloads or artifacts.
- The JSON diff implementation treats lists as atomic, so `outputs_summary` list
  changes appear as a single path change (for example, `/orbit_pass/cases`).
