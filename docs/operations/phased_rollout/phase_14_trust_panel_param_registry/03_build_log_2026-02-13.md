# Build Log

## Metadata
- Work item ID: PT-PHASE-14
- Title: Trust panel v0.2 (parameter registry + units/ranges + validation surface)
- Date: 2026-02-13

## 1) Outcome

Implemented a backend-owned **kinds/parameter registry** and exposed it via the
local API server. Updated the web editor to consume this registry and render a
"trust panel" for selected nodes showing:
- units
- defaults
- ranges
- enums
- CLI-only / API-disabled status (e.g., Touchstone import)

This makes parameter meaning explicit without moving physics logic into the UI.

## 2) Code changes

### 2.1 Backend registry
- Added:
  - `photonstrust/registry/__init__.py`
  - `photonstrust/registry/kinds.py`
- Registry content includes:
  - QKD kinds: `qkd.source`, `qkd.channel`, `qkd.detector`, `qkd.timing`, `qkd.protocol`
  - PIC kinds: `pic.waveguide`, `pic.coupler`, `pic.phase_shifter`, `pic.ring`, `pic.grating_coupler`, `pic.edge_coupler`, `pic.touchstone_2port`
  - Param schema entries: type/unit/default/min/max/enum/description + optional `applies_when`.
  - Availability flags: `api_enabled` and `cli_enabled` per kind.

### 2.2 API endpoint
- Modified:
  - `photonstrust/api/server.py`
- Added:
  - `GET /v0/registry/kinds`
    - returns `registry` + `registry_hash` + provenance metadata.

### 2.3 Web editor trust panel + palette sourcing
- Modified:
  - `web/src/photontrust/api.js` (added `apiGetKindRegistry`)
  - `web/src/App.jsx`
  - `web/src/App.css`
- Behavior:
  - `Ping` now fetches both `/healthz` and `/v0/registry/kinds`.
  - Palette titles prefer backend registry titles when available.
  - Inspector shows `KindTrustPanel` above raw JSON params editor.
  - Quick-edit controls write back into node `params` while keeping raw JSON
    editing for advanced cases.

### 2.4 Optional API test extension
- Modified:
  - `tests/api/test_api_server_optional.py`
- Added:
  - smoke coverage for `GET /v0/registry/kinds`

## 3) Notes / decisions

- UI defaults remain conservative: it does not automatically inject all backend
  defaults into node params (to avoid freezing derived defaults). The trust
  panel exposes defaults and constraints so users can make informed edits.
- Touchstone is explicitly marked as `api_enabled: false` in the registry and
  continues to be blocked in API execution.
