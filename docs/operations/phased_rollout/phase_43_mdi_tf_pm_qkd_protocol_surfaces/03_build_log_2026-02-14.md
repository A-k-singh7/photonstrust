# Phase 43: MDI-QKD + TF/PM-QKD Protocol Surfaces (Build Log)

Date: 2026-02-14

## Implemented

- Added QKD protocol-family dispatch keyed off `scenario.protocol.name`:
  - Direct-link baseline: `BBM92` / `E91` (existing entanglement-based model)
  - Relay protocols:
    - `MDI_QKD` (Xu et al., arXiv:1305.6965)
    - `PM_QKD` and `TF_QKD` (Ma et al., arXiv:1805.05538; TF surface aliases PM model)
- Added shared result type to avoid circular imports:
  - `photonstrust/photonstrust/qkd_types.py`
- Added protocol implementation package:
  - `photonstrust/photonstrust/qkd_protocols/common.py`
  - `photonstrust/photonstrust/qkd_protocols/mdi_qkd.py`
  - `photonstrust/photonstrust/qkd_protocols/pm_qkd.py`
- Refactored QKD entry point to use shared result type and dispatch:
  - `photonstrust/photonstrust/qkd.py`
- Extended protocol registry and config schema enums to expose new protocol names + parameters:
  - `photonstrust/photonstrust/registry/kinds.py`
  - `photonstrust/schemas/photonstrust.config.demo1.schema.json`
- Updated Reliability Card notes to surface relay-protocol assumptions:
  - `photonstrust/photonstrust/report.py`
- Tests:
  - Scoped PLOB sanity gate explicitly to `protocol.name: BBM92`:
    - `photonstrust/tests/test_qkd_plob_bound.py`
  - Added relay-protocol surface sanity tests:
    - `photonstrust/tests/test_qkd_relay_protocol_surfaces.py`

## Implementation notes

- Relay protocols currently support `channel.model: fiber` only.
- `TF_QKD` is currently a labeling surface over the same Appendix-B PM-QKD analytical model; if we later introduce TF-specific variants, the branch point is in `photonstrust/photonstrust/qkd_protocols/pm_qkd.py`.
