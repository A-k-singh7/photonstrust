# Build Log

## Metadata
- Work item ID: PT-PHASE-11
- Date: 2026-02-13

## Summary
Implemented OrbitVerify mission templates v0.1 for time-segmented pass envelopes.

This phase adds a new execution path that:
- treats a pass as explicit samples over time (no orbit propagation),
- runs the QKD point model for each sample,
- exports free-space contributor decomposition per sample, and
- writes a versioned pass results artifact plus a simple HTML report.

## Code changes

### Schemas
- Added:
  - `schemas/photonstrust.orbit_pass_results.v0.schema.json`

### OrbitVerify pass execution
- Added:
  - `photonstrust/orbit/__init__.py`
  - `photonstrust/orbit/pass_envelope.py`
    - `simulate_orbit_pass(config)` -> pass results dict
    - `run_orbit_pass_from_config(config, output_root)` -> writes artifacts:
      - `orbit_pass_results.json`
      - `orbit_pass_report.html`

### CLI integration
- Updated:
  - `photonstrust/cli.py`
    - `photonstrust run <config.yml>` now detects `orbit_pass` and runs the
      orbit pass runner, writing `orbit_pass_run.json` at the output root.

### Scenario templates
- Added:
  - `configs/demo11_orbit_pass_envelope.yml`

### Tests
- Added:
  - `tests/test_orbit_pass_envelope.py`
    - known-sense checks:
      - elevation improves key rate (all else equal)
      - background worsens QBER and key rate (all else equal)
      - best-case >= worst-case at fixed samples
    - schema validation against the new results schema

## Notes and limitations
- v0.1 uses an explicit pass envelope (samples + dt); it is not an orbit propagator.
- Weather/availability modeling is not included in this phase.
- The pass report is intentionally minimal; it is a trust artifact starter, not
  a final mission-review template.
