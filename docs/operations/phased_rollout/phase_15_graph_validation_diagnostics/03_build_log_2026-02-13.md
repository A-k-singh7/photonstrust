# Build Log

- Title: Graph validation + structured diagnostics (params, ports, kind support)
- Date: 2026-02-13

## 1) Outcome

Implemented backend-owned graph semantic validation that produces structured,
machine-readable diagnostics for:
- parameter schemas (type, enum membership, numeric ranges, unknown params)
- `applies_when` mismatches (best-effort warning)
- PIC kind support checks (unsupported kinds are explicit errors)
- PIC edge port validation (`from_port`/`to_port` must exist for the component)
- QKD graph required kind completeness (missing/duplicate required nodes)

Exposed this through the local API and surfaced diagnostics in the web editor
compile tab.

## 3) Key trust decision

PIC port validation uses the **engine truth** (`component_ports(kind)`), not UI
assumptions. This prevents silent “dropped signal” behavior due to incorrect
port names in graph edges.
