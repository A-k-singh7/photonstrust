# Phase 35 - Workflow Replay + Evidence Bundle Export - Build Log

Date: 2026-02-14

## Summary
Added two control-plane capabilities for trustable review workflows:
- A portable evidence bundle export (`.zip`) for runs, including workflow child runs by default.
- Workflow replay from recorded request inputs, with explicit provenance linking.

Also added Run Browser UX so workflow runs are navigable (bundle download + child-run links + replay action).

## Changes (By File)

### Backend
- `photonstrust/photonstrust/api/server.py`
  - Added `GET /v0/runs/{run_id}/bundle`:
    - builds a deterministic zip with stable ordering and fixed zip timestamps
    - includes `bundle_manifest.json` (sha256 per file) and `README.md`
    - includes workflow child runs when the selected run is a workflow
    - uses safe artifact resolution under run directories only
  - Updated `POST /v0/pic/workflow/invdesign_chain`:
    - writes `workflow_request.json` into the workflow run directory
    - adds `workflow_request_json` to the workflow run manifest artifacts
    - supports `replayed_from_run_id` provenance field
  - Added `POST /v0/pic/workflow/invdesign_chain/replay`:
    - loads `workflow_request.json` from an existing workflow run
    - re-executes the chain
    - records `replayed_from_run_id` in the new workflow run

### Web
- `photonstrust/web/src/photontrust/api.js`
  - Added `apiReplayPicInvdesignWorkflowChain()`.
- `photonstrust/web/src/App.jsx`
  - Added bundle URL helper (`/v0/runs/{run_id}/bundle`) links for workflow results.
  - Runs -> Manifest: added a Workflow Chain callout (bundle download + child-run links + replay button).

### Tests
- `photonstrust/tests/api/test_api_server_optional.py`
  - Added bundle export coverage:
    - asserts zip contains workflow manifest + artifacts and key child run artifacts.
  - Added replay coverage:
    - asserts replay creates a new workflow run and records `replayed_from_run_id`.
  - Updated workflow chain test to assert `workflow_request.json` is written.

## Notes / Decisions
- Bundle export is designed to be safe by construction: only manifest-declared artifacts under run dirs can be included.
- Bundle zip entries use a fixed timestamp to reduce non-content diffs between builds.
- Replay depends on `workflow_request.json`, so old Phase 34 workflow runs created before this phase cannot be replayed (API returns a clear error).
