# Phase 24 — Build Log — 2026-02-13

## Summary
Added a v0.2 seam for GDS ingestion using an optional `gdstk` dependency and formalized a JSON schema for extracted parallel-run layout features.

## Code Changes

### Added
- `photonstrust/layout/gds_extract.py`
  - Optional `gdstk` backend for extracting PhotonTrust `routes[*]` from GDS PATH spines.
  - Optional conversion of axis-aligned rectangle polygons into 2-point routes.
- `schemas/photonstrust.layout_parallel_runs.v0_1.schema.json`
- `tests/test_layout_features_schema.py`
- `tests/test_gds_extract_optional.py`

### Modified
- `pyproject.toml`
  - Added optional extra: `layout = ["gdstk>=0.9"]`

## Notes / Decisions
- This phase intentionally does **not** implement full signoff-grade GDS extraction:
  - no hierarchy traversal / reference transforms (v0.2),
  - no curved path decomposition for the Manhattan-only route extractor,
  - polygons are only supported if they are axis-aligned rectangles.
