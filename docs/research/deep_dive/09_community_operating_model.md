# Community Operating Model

This document defines maintainership and contribution practices.

## Governance
- Core maintainers own roadmap and release approvals
- Contributor guidelines define review and testing expectations

## Contribution flow
- Issue templates: bug, feature, benchmark request
- PR template with reproducibility and test checklist

## Triage policy
- Severity classes for correctness and reliability issues
- SLA for first response on high-impact issues

## Documentation policy
- Every major feature ships with docs updates
- Keep research docs and implementation in sync

## Definition of done
- Contributor docs published.
- Issue/PR templates active and used.
